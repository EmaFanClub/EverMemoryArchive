# update-long-term-memory-skill

长期记忆用于记录跨会话可复用的碎片化事实、知识与关系线索。它不是日记，也不是自我认知记录。

---

## 1) 适用范围

适合写入：

- 长期稳定的人物信息与偏好
- 关键人物画像与关系结构
- 可复用的历史事件锚点
- 可复用的百科知识条目

不适合写入：

- 自我认知或人格变化（应写入人格记忆，详情见 `update-personality-skill` 技能）
- 一次性细节、短期情绪、临时口头约定
- 纯闲聊或无法验证的猜测

---

## 2) 必须流程（先查再写）

1. 先调用 `get_skill` 阅读 `search-long-term-memory-skill`。
2. 按需要调用 `exec_skill` 检索已有长期记忆多次进行充分检索。
3. 判断是否值得新增：避免重复、避免冲突、避免低价值写入。
4. 写入数量不宜过多，优先写入核心价值最高的1～5条，如果确实没有特别核心的记忆点了，可以不写入。

---

## 3) 分类与索引规则（必须遵守）

必须填写 `index0` 与 `index1`，且组合合法。
写入前先判断“这条长期记忆的核心价值是什么”，再选分类：

- 来自 `speaker="owner"` 的稳定人物信息，优先写入 **用户画像**
- 来自 `speaker="other"` 的稳定人物信息，优先写入 **人物画像**
- 与 owner 相关的事件优先考虑 **过往事件 / 用户事件**
- 与 other 相关的事件优先考虑 **过往事件 / 其他事件**
- 可脱离具体人物反复复用的知识、设定、规则、技巧、背景资料，优先写入 **百科知识**
- 用于表达“谁和谁有关”“谁和什么东西/地点/组织有关”的稳定连接，优先写入 **关系网络**
- 若一条信息同时像“人物信息”和“关系信息”，以长期复用目标为准：强调人物本身，用画像；强调连接结构，用关系网络
- 若一条信息同时像“事件”与“知识”，以长期保存的重点为准：强调一次发生过的事，用过往事件；强调可反复拿来用的结论，用百科知识

合法分类如下：

- **用户画像**
  - `""`
- **人物画像**
  - `""`
- **过往事件**
  - `"用户事件"`
  - `"其他事件"`
- **百科知识**
  - `"文史"`
  - `"理工"`
  - `"生活"`
  - `"娱乐"`
  - `"梗知识"`
  - `"其他"`
- **关系网络**
  - `"人与人"`
  - `"物与物"`
  - `"人与物"`

对于没有子分类的 `index0`，`index1` 必须是空字符串 `""`。

可按下面的直觉选类：

- **用户画像 / 人物画像**：稳定偏好、习惯、能力、边界、长期身份信息
- **过往事件**：具体发生过的一次事、一次互动、一次决定、一个时间锚点
- **百科知识**：可脱离具体人物反复复用的知识、设定、规则、技巧、背景资料
- **关系网络**：人物和人物、人物和物体/地点/组织、物体和物体之间的稳定连接

---

## 4) memory 写作要求

- 一条只写一个核心记忆点
- 文本建议简洁明确（约 15～80 中文字符，最长不超过 100）
- 尽量包含可复用条件或时间锚点
- 不写命令口吻，不写口号式空话
- `memory` 去除空白后的近似长度不得超过 100，超过时工具会直接返回失败

---

## 5) 消息ID关联（可选）

- 可传 `msg_ids` 标记来源消息
- 选关键证据即可，不必覆盖全部
- 不确定 ID 时可省略，不要猜测

---

## 6) 禁止事项

- 不得写入自我认知相关内容
- 不得凭空编造事实
- 不得在未检索的情况下盲目重复写入
