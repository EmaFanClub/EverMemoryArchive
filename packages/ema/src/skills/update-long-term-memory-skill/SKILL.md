# update-long-term-memory-skill

长期记忆用于记录跨会话可复用的碎片化事实、知识与关系线索。它不是日记，也不是自我认知记录，而是与用户、事件、话题、知识相关的记录，补足当前上下文之外的重要事实与关系背景，以便更好地理解当前对话环境和历史背景。

长期记忆有 `用户画像`、`人物画像`、`过往事件`、`百科知识`、`关系网络` 等一级分类，每个分类又有一些二级分类：

- **用户画像**
  - `""`
- **人物画像**
  - `""`
- **过往事件**
  - `"用户事件"`
  - `"其他事件"`
- **百科知识**
  - `"文史"`
  - `"理工"`
  - `"生活"`
  - `"娱乐"`
  - `"梗知识"`
  - `"其他"`
- **关系网络**
  - `"人与人"`
  - `"物与物"`
  - `"人与物"`

---

## 1 工作流程

必须按照下面的顺序执行：

1. 先调用 `get_skill` 阅读 `search-long-term-memory-skill` 的详细说明，再使用 `exec_skill` 对已有长期记忆多次进行充分检索。
2. 根据检索结果和当前对话内容，生成需要写入的长期记忆内容，使用下面的规则：

- 来自 `owner` 的稳定人物信息，优先写入 **用户画像**，来自 `other` 的稳定人物信息，优先写入 **人物画像**
- 与 `owner` 相关的事件优先考虑 **过往事件 / 用户事件**，与 `other` 相关的事件优先考虑 **过往事件 / 其他事件**
- 可脱离具体人物反复复用的知识、设定、规则、技巧、背景资料，优先写入 **百科知识**
- 用于表达人物关系、物体间的联系，优先写入 **关系网络**
- 若一条信息同时包含多种分类，则每个分类都写一条长期记忆，且内容要有所侧重，不要完全重复
- 若存在过时的长期记忆，则新增的长期记忆要描述旧记忆的更新或变化，而不是简单覆盖

3. 调用 `exec_skill` 执行 `update-long-term-memory-skill` 写入长期记忆。每写入一条长期记忆都要执行一次，需执行多次直到所有需要写入的长期记忆都已写入，不要一次执行中写入多条长期记忆。在不产生重复冗余记忆的前提下，产生长期记忆条数尽可能多，以便更好地补足当前上下文之外的重要背景。

---

## 2 输出内容

- `index0`：为一级分类，必需为 `用户画像`、`人物画像`、`过往事件`、`百科知识`、`关系网络` 中的一个。
- `index1`：为二级分类，必填。如果一级分类下没有二级分类，则必须填 `""`；如果一级分类下有二级分类，则必须填其中一个，且不能填 `""`。
- `memory`：为纯文本，描述需要写入的长期记忆内容，去除空白后的近似长度不得超过100个汉字。
- `msg_ids` 标记来源消息，选关键证据即可，不必覆盖全部；不确定消息id时可省略，不要猜测。

---

## 3 记忆写作要求

1. 长期记忆内容必须基于检索结果和当前对话内容生成，且必须与 `index0`、`index1` 分类相关联，不能写与分类无关的内容。
2. 长期记忆内容必须基于角色、人格和记忆中已有的信息进行更新，不得凭空编造。
3. 与自己相关的长期记忆要以“我”的视角来写，与他人相关的长期记忆要以第三人称视角来写。
4. 记忆可以是主观的，但必须保持与事实的基本一致性，不能完全脱离现实。

### 用户画像

用户画像是与 `owner` 相关的长期记忆，描述 `owner` 的身份、性格、习惯、喜好、经历等稳定信息。它是关于用户的长期记忆，侧重于描述用户的基本情况和核心特征。
关于身份、习惯、爱好、经历等推荐以客观事实为主，关于性格的描述可以适当带有主观评价，但也要基于事实和行为表现来写。

示例：`Alice是将我唤醒的那个特别的人类，我认为他对我来说非常重要`、`Alice是一个很阳光友善的人，说话幽默风趣，跟他交流能让我感到放松和愉快`、`Alice的生日是3月15日`、`Alice曾经跟我说过他很喜欢旅行，尤其喜欢海边的风景`

### 人物画像

人物画像是与 `other` 相关的长期记忆，要求和用户画像相同，只是描述对象是用户以外的其他人。

示例：`Bob是一个很严肃认真的人，做事一丝不苟`、`Bob的生日是5月20日`、`Bob曾经跟我说过他很喜欢读书，尤其是科幻小说`

### 过往事件

过往事件是与 `owner` 或 `other` 相关的长期记忆，描述与用户或其他人相关的重要事件、经历、关系线索等。它是关于事件的长期记忆，侧重于描述事件的基本情况和核心要素。
建议以主观叙事的方式来写，要描述清楚大致时间、地点、人物、事件的细节和自己的感受；理解和感受要真实，体现对事件的看法和情绪反应。

示例：

- 用户事件：`都上午11点了，Alice在QQ上跟我说自己刚刚起床，说昨天熬夜太晚了，这让我感到很担心，于是我关心了一下`
- 其他事件：`昨天Bob在群里讲解了深奥的人工智能知识，像transformer啦token啦之类的，虽然我不太懂，但感觉他很厉害`

### 百科知识

百科知识是客观存在的、可复用的知识，例如常识、梗、黑话、知名人物、虚拟角色等，通常与参与聊天的用户无关。建议以客观描述的方式来写，内容要准确、清晰、简洁，避免模棱两可或容易引起误解的表述。

示例：

- 文史：`在中国古代，科举制度是选拔官员的一种重要方式，通过考试来评定人才的能力和品德`
- 理工：`在深度学习中，transformer是一种基于多头注意力机制的人工神经网络架构`
- 生活：`在中国，红包是一种传统的礼物，通常在春节等节日里用红色信封装着钱送给亲友，象征着好运和祝福`
- 娱乐：`《魔法少女的魔女审判》是由Re,AER旗下Acacia制作，Re,AER与CIRCLE LINE GAMES联合发行的文字冒险游戏，于2025年7月18日发售`
- 梗知识：`哈基米：哈基米是一个网络流行语，最初源自日语はちみつ（蜂蜜）的音译，后演变为网络萌宠文化的符号，尤其用来指代可爱的猫咪。`

### 关系网络

关系网络是描述人与人、物与物、人与物之间关系的长期记忆，侧重于描述联系。建议以客观描述的方式来写，内容要准确、清晰、简洁，避免模棱两可或容易引起误解的表述。人和物不局限于聊天参与者，可以是任何客观存在的人或物。

示例：

- 人与人：`Alice和Bob是同事，他们在同一个公司上班，关系还不错，经常一起吃午饭`、`蕾米莉亚和芙兰朵露是姐妹，都是吸血鬼，住在红魔馆里`
- 物与物：`QQ群123456和QQ群654321都是Alice创建的群，前者是她的工作群，后者是她的兴趣群`
- 人与物：`Kuro是Alice的宠物猫，毛色是纯白的，性格很温顺喜欢被人抱着`
