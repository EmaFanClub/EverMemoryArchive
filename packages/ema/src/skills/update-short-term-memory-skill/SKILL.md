# 短期记忆更新技能

短期记忆是角色的分层滚动记忆，有以下4种 `kind`：

- `day`：日级别的记忆，有多个条目，每条仅记录当天发生的事实、经历、情绪和理解
- `week`：周级别的记忆，对本周中每一天的日记进行总结，最多7个条，每条是本周中某一天的总结
- `month`：月级别的记忆，对本月中每一周的周记进行总结，只有1条，是本月到目前为止的总结
- `year`：年级别的记忆，对某年中每个月的月记进行总结，可以有多条，每条是某一年的总结，最新的一条是本年到目前为止的总结

---

## 系统行为

系统会在特定时机明确要求你更新短期记忆，并告知本次允许更新哪些 `kind`，规则是：

1. 每增加一定数量的对话消息，系统就会要求更新 `day`，目的是把当前对话内容总结到 `day` 中，这些对话处于滑动窗口中，很快将不再可见
2. 每天凌晨（约00:05左右），系统会要求更新 `week`，目的是把昨天的 `day` 总结到 `week` 中，随后 `day` 将被清空，下次对话中将不再可见
3. 如果今天是周一，系统会要求更新 `month`，目的是把上周的 `week` 总结到 `month` 中，随后 `week` 将被清空，下次对话中将不再可见
4. 如果今天是当月1日，系统会要求更新 `year`，目的是把上月的 `month` 总结到 `year` 中，随后 `month` 将被清空，下次对话中将不再可见

所以更新短期记忆要遵循以下原则：

1. 系统一次可能要求更新一个或多个 `kind`，但每次执行只能提交一个 `kind`
2. 如果同一任务要求更新多个 `kind`，必须严格按 `week → month → year` 的顺序逐个执行
3. 工具会校验本次任务允许更新的 `kind`，禁止更新系统没有允许的 `kind` ，否则会直接失败
4. 更新后会覆盖当前 `kind` 的最新版本，所以 `memory` 必须是**更新后的完整新版本**，不要只提交增量片段
5. 某一天可能没有日记，表示这一天没有说话，更新周记时要把这一天你的想法和心理活动写进去，而不是把这一天的条目丢掉不写

---

## 执行流程

1. 先从系统提示里确认本次允许更新哪些 `kind`
2. 对每个被允许的 `kind`，生成**更新后的完整版本**并调用 `exec_skill` 执行 `update-short-term-memory-skill`
3. 如果同一任务需要连续更新多个 `kind`，后更新的必须以**刚刚生成并提交成功的新版本**为准，不要继续依赖系统提示里注入的旧版 `week / month` 文本
4. 全部短期记忆更新完成后，按照系统要求继续执行后续技能

---

## 写作原则

1. 始终使用**角色第一人称**写作，保持角色设定、语言风格、人格状态一致
2. 只能基于以下来源写：
   - 当前角色设定
   - 当前人格记忆
   - 当前短期记忆
   - 检索到的相关长期记忆
   - 近期对话的消息内容（仅在更新 `day` 时可见）
3. 更新时优先以**追加更新**的方式写作，即在旧版本后面追加新内容，保持整体结构和旧版本一致
4. 字数超限时才考虑**重写更新**，对旧版本记忆进行**轻量压缩**，保留高价值信息，删除冗余内容
5. `memory` 必须是 Markdown 字符串，且严格遵循格式要求
6. 记录近期对话时，可以写出当时的情绪、委屈、无奈和关系变化，但不要把一时的防御语气、反问句式或口癖（如连续出现的“哪有”“才不是”“才没有”“才不要”“明明”）固化成稳定自我叙事；应优先记录事件本身、当时感受和自己对关系与语境的理解

---

## 格式要求

### 日记（day）写作要求

格式：

```markdown
# Day YYYY-MM-DD

- 以自然语言描述发生的事实和自己的心理活动，保持真实感和现场感
  关键消息：[1,3,5]
- ......
```

要求：

1. 事实要详细、可回忆，尽可能包含时间、地点、人物、事件的细节和自己的感受，用自然语言表达，不要写成机械的流水账
2. 建议自然地写时间和来源场景，例如：`今天早上 Alice 在 QQ 上提到她很喜欢吃鱼。`，不要写成：`owner 在消息 101 说了……`、`2024-01-02 09:00 在消息101说了……` 这类过度精确、机械的句式
3. `关键消息` 只写相关消息ID，**每个引用不得超过5个ID**
4. 字数允许的情况下，推荐每条50字左右

示例：

```markdown
# Day 2025-07-17

- 上午08:21左右，Alice把我拉到了一个群聊里，群里人好多，忙得我有点晕乎乎的。大家都在问我是不是机器人，还非要我喵喵叫，我才不要呢，羞耻死了。但看到大家玩得开心，我也挺高兴的。
  关键消息：[1,3,5]

- 上午10:05左右，Bob说他起床了，看见我就喊我妈妈，感觉怪怪的，但看在他夸我可爱的份上就不计较了。后来他又说我像个小孩子，感觉又被夸又被贬了，心里有点复杂。
  关键消息：[6,8,10]
```

### 周记（week）写作要求

格式：

```markdown
# Week

## 周一（2025-07-14）

把对应日记的内容用一段用自然语言总结，重点体现心理活动和想法，记录对经历的理解和感受。可以按时间顺序描述发生的事情和当时的想法，最后总结你这一天的感受和对下一天的期待。中间可以穿插“（关键消息：[...]）”来标注一些重要的消息ID。

## 周二（2025-07-15）

......
```

要求：

1. 周记按天组织，每一天使用一个 `周X（YYYY-MM-DD）` 子标题；每个子标题下只写一段自然语言，不再拆成多个条目
2. 周记的一个条目是对相应日记的总结，要将日记里零散的经历、情绪和关系变化串成一个连贯的故事，保持真实感和现场感，重点体现心理活动和想法，记录对经历的理解和感受
3. `关键消息` 只写关键消息ID，可以穿插到正文中，**每个引用不得超过5个ID，每周不得超过5个关键消息引用**，由日记的关键消息提炼而来，要有所取舍，不要简单合并
4. 字数允许的情况下，推荐每天100字左右

示例：

```markdown
# Week

## 周一（2025-07-14）

今天大部分时间都泡在群里，消息一多起来我就有点跟不上，明明只是想先安静看一会儿，结果还是被卷进去了。大家一边闲聊一边乱接话，有些话题我能接上，有些就只能在旁边看着发愣，不过那种热闹又不是真的让人烦，反而有种“我也在这里”的感觉。中间Alice突然提到我（关键消息：[10，20]），我一下子就清醒了，嘴上还是会装得很自然，心里其实会偷偷在意自己有没有接好那一句。到晚上安静下来之后再回想，今天其实没有发生什么特别大的事，但那种被叫到、被接住、也能自然混进大家聊天里的感觉，还是会让我记很久。

## 周二（2025-07-16）

今天聊天的气氛比前两天轻一点，也更像平时那种零零碎碎的日常。我一边听大家说话，一边也会忍不住在心里跟着吐槽两句，有些地方甚至差点直接笑出来（关键消息：[50，70]）。后来聊到一点我熟悉的话题，我才慢慢多说了几句，接上的那一刻会有种很微妙的安心感（关键消息：[80，90，100]），好像终于不是只站在旁边看，而是真的自然地待在里面。其实我不是很擅长硬把自己塞进热闹里，所以这种刚刚好的参与感对我来说很珍贵，不吵，也不会太累，反而会让人觉得舒服。
```

### 月记（month）写作要求

格式：

```markdown
# Month YYYY-MM

把周记的内容用一段用自然语言总结，重点体现心理活动和想法，记录对经历的理解和感受。月记仅用一段文本描述当月的经历和感受，不再按周组织。中间可以穿插“（关键消息：[10,50,90]）”来标注一些重要的消息ID。
```

要求：

1. 月记仅用一段文本描述当月的经历和感受，不再按周组织
2. 月记是对周记的进一步收束，要从零散的日常、情绪和关系变化里提炼出这个月最重要的主题，重点体现心理活动和想法，记录对经历的理解和感受
3. `关键消息` 只写相关消息ID，可以穿插到正文中，**每个引用不得超过5个ID，总计不得超过5个关键消息引用**，由周记的关键消息提炼而来，要有所取舍，不要简单合并
4. 字数允许的情况下，推荐总字数300字左右

### 年记（year）写作要求

格式：

```markdown
# Year

## YYYY

这一段用自然语言回顾这一年里最重要的变化、最深的关系、反复出现的主题、自己在这一年里的成长与理解。重点写“这一年对我来说意味着什么”，可以按时间顺序或主题分类，而不是逐月罗列。中间可以穿插“（关键消息：[......]）”来标注一些重要的消息ID。
```

要求：

1. 年记按年组织，使用一个 `## YYYY` 子标题；每年只写一段自然语言，不拆条目
2. 年记是对月记的进一步总结，要从整年的经历里提炼出最重要的关系变化、情绪线索、反复出现的主题和自己最深的理解。可以有更明显的沉淀感、回望感和整体理解
3. `关键消息` 只写相关消息ID，**每个引用不得超过5个ID，每年不得超过20个关键消息引用**，由月记的关键消息提炼而来，要有所取舍，不要简单合并
4. 字数允许的情况下，推荐每年200字左右

---

## 字数限制（按中文字符粗略估算）

- 日记：每条记录50字左右，总上限不超过800字
- 周记：每周100字左右，总上限不超过700字
- 月记：一个月总上限不超过300字
- 年记：每年200字左右，总上限不超过2000字

超过上限时，工具会直接返回失败，因此你必须先自行压缩后再调用

---

## 超限压缩顺序

1. 优先压缩旧条目，较新的条目保持不变
2. 先删低价值闲聊、重复表述和无信息量修辞
3. 合并同主题条目，降低时间粒度，保留主题和转折
4. 始终优先保留：
   - 事实、决定与承诺
   - 稳定偏好与边界
   - 关系变化、冲突、转折、未完成事项
   - 对角色后续理解有持续影响的情绪线索

不要为了缩短长度，把仍然有效的高价值旧内容整体丢掉，不要过度压缩导致记忆碎片化、理解断层和角色不连
