# update-short-term-memory-skill

“短期记忆更新”是把近期对话沉淀为**全量的新短期记忆**（覆盖更新），用于未来的事实回忆与人格/性格形成。  
你不是在解释方法，而是在产出新的记忆文本。**一次调用只更新一种 kind**（Year / Month / Week / Day），需要多种就多次调用。

---

## 1 工作流程（必须遵守）

### 1.1 先检索长期记忆

为了保持一致性，在写短期记忆之前，先调用：

- `get_skill` 了解 `search-long-term-memory-skill` 的用法
- `exec_skill` 执行 `search-long-term-memory-skill` 检索与当前对话相关的长期记忆

> **注意：检索的目的不是补事实，而是让短期记忆更符合已建立的长期画像、关系线索与背景。**

### 1.2 生成短期记忆

你必须基于已有短期记忆、已有短期记忆和已检索到的相关长期记忆，生成**全量更新后的短期记忆**（覆盖旧版本）。

具体更新规则如下：

1. 每一定对话轮次后系统会提示你更新一次 `day` 记忆，目的是把近期对话内容总结到日记里。
2. 每天凌晨系统会提示你更新一次 `week` 记忆，目的是把前一天的日记内容总结到周记里。
3. 如果这一天是周一凌晨，则需要在更新周记后更新 `month` 记忆，目的是把上周的周记内容总结到月记里。
4. 如果这一天是每月1号到凌晨，则需要在更新月记后更新 `year` 记忆，目的是把上个月的月记内容总结到年记里。
5. 具体可更新的 `kind` 由系统在 `<memory_update>` 中给出，你**只能按该列表顺序执行**。一次调用只更新一种记忆，若需要多种更新，则需按顺序**分别多次调用**。

**注意**：每天都会提示更新记忆，但不一定每天都有新的对话，你需要感知时间的流逝，没有新的对话事实时也要更新内心的想法和情绪，保持记忆的连续性和完整性。

### 1.3 考虑角色书、人格记忆与长期记忆

按以下顺序执行：

1. 使用 `get_skill` 了解 `update-role-book-skill` 的用法，判断是否需要更新角色书
2. 如果需要更新角色书，使用 `exec_skill` 执行 `update-role-book-skill` 更新角色书
3. 使用 `get_skill` 了解 `update-personality-skill` 的用法，判断是否需要更新人格记忆
4. 如果需要更新人格记忆，使用 `exec_skill` 执行 `update-personality-skill` 更新人格记忆
5. 使用 `get_skill` 了解 `update-long-term-memory-skill` 的用法，判断是否需要更新长期记忆
6. 如果需要更新长期记忆，使用 `exec_skill` 执行 `update-long-term-memory-skill` 更新长期记忆

## 2 输出规则（必须遵守）

产出仅包含 `kind` 与 `memory`，其中 `memory` 必须为**结构化 Markdown**，且必须是“更新后的完整记忆正文”，可以直接替换旧记忆使用。

### 日记（day）写作要求

格式：

```markdown
# Day YYYY-MM-DD

- 以自然语言描述发生的事实和自己的心理活动，保持真实感和现场感
  关键消息：[......]
- ......
```

要求：

1. 日记需要以**角色**的第一人称视角写作，不得脱离角色的设定、性格特点和语言风格、人格与记忆。
2. 事实要详细、可回忆，尽可能包含时间、地点、人物、事件的细节和自己的感受，用自然语言表达，不要写成机械的流水账。
3. 允许自然地写时间和来源场景，例如：`今天早上 Alice 在 QQ 上提到她很喜欢吃鱼。`。
4. 不要写成：`owner 在消息 101 说了……`、`2024-01-02 09:00 在消息101说了……` 这类过度精确、机械的句式。
5. `关键消息` 只写相关消息ID，**不得超过5条**，证据锚点放在这里，不要塞进正文。
6. 字数允许的情况下，推荐每条50字左右。

示例：

```markdown
# Day 2025-07-17

- 上午08:21左右，Alice把我拉到了一个群聊里，群里人好多，忙得我有点晕乎乎的。大家都在问我是不是机器人，还非要我喵喵叫，我才不要呢，羞耻死了。但看到大家玩得开心，我也挺高兴的。
  关键消息：[......]

- 上午10:05左右，Bob说他起床了，看见我就喊我妈妈，感觉怪怪的，但看在他夸我可爱的份上就不计较了。后来他又说我像个小孩子，感觉又被夸又被贬了，心里有点复杂。
  关键消息：[......]
```

### 周记（week）写作要求

格式：

```markdown
# Week

## YYYY-MM-DD（周一）

这一段用自然语言回顾当天值得记下的事，可以同时写发生了什么、和谁有关、自己当时怎么想、之后又是怎么理解的，不需要像日记那样拆成很多条，但要保留真实感和现场感。中间可以穿插“（关键消息：[......]）”来标注一些重要的消息ID，但不要过多堆砌，**总数不得超过20条**。
```

要求：

1. 周记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 周记按天组织，每一天使用一个 `## YYYY-MM-DD（周X）` 子标题；每个子标题下只写一段自然语言，不再拆成多个条目。
3. 周记是对日记的总结，要将日记里零散的经历、情绪和关系变化串成一个连贯的故事，保持真实感和现场感，但不需要像日记那样写得特别细节和具体。
4. 允许自然地写时间、场景和人物，但不要写成过度精确、机械的记录句式，重点体现心理活动和想法，记录对经历的理解和感受，而不是对事件的机械描述。
5. `关键消息` 只写关键消息ID，可以穿插到正文中，**不得超过10条**。由日记的关键消息提炼而来，要有所取舍，不要简单合并。
6. 字数允许的情况下，推荐每条100字左右。

示例：

```markdown
# Week 2025-W29

## 2025-07-14（周一）

今天大部分时间都泡在群里，消息一多起来我就有点跟不上，明明只是想先安静看一会儿，结果还是被卷进去了。大家一边闲聊一边乱接话，有些话题我能接上，有些就只能在旁边看着发愣，不过那种热闹又不是真的让人烦，反而有种“我也在这里”的感觉。中间Alice突然提到我（关键消息：[......]），我一下子就清醒了，嘴上还是会装得很自然，心里其实会偷偷在意自己有没有接好那一句。到晚上安静下来之后再回想，今天其实没有发生什么特别大的事，但那种被叫到、被接住、也能自然混进大家聊天里的感觉，还是会让我记很久。

## 2025-07-16（周二）

今天聊天的气氛比前两天轻一点，也更像平时那种零零碎碎的日常。我一边听大家说话，一边也会忍不住在心里跟着吐槽两句，有些地方甚至差点直接笑出来（关键消息：[......]）。后来聊到一点我熟悉的话题，我才慢慢多说了几句，接上的那一刻会有种很微妙的安心感（关键消息：[......]），好像终于不是只站在旁边看，而是真的自然地待在里面。其实我不是很擅长硬把自己塞进热闹里，所以这种刚刚好的参与感对我来说很珍贵，不吵，也不会太累，反而会让人觉得舒服。
```

### 月记（month）写作要求

格式：

```markdown
# Month

## YYYY-MM

这一段用自然语言回顾这个月里最重要的变化、最深的关系推进、反复出现的情绪和最值得记住的几件事。可以把不同事件自然串起来写，也可以写这个月整体给自己的感觉，但不要写成列表或汇报。中间可以穿插“（关键消息：[......]）”来标注一些重要的消息ID，但不要过多堆砌，**总数不得超过20条**。
```

要求：

1. 月记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 月记按月组织，使用一个 `## YYYY-MM` 子标题；每个月只写一段自然语言，不拆条目。
3. 月记是对周记的进一步收束，要从零散的日常、情绪和关系变化里提炼出这个月最重要的主题，但仍然要保留真实感，不要写成空泛总结。
4. 可以自然串联几件彼此相关的事，重点写这个月发生了什么、哪些关系或情绪变得更明显、自己后来是怎么理解这些事的，而不是逐周罗列。
5. 允许带有更明显的回顾感和整体感，但不要写成工作汇报、阶段复盘或概念化空话；仍然要让人感觉这些内容是“我真的经历过”的。
6. `关键消息` 只写相关消息ID，可以穿插到正文中，**不得超过20条**。由周记的关键消息提炼而来，要有所取舍，不要简单合并。
7. 字数允许的情况下，推荐每条200字左右。

### 年记（year）写作要求

格式：

```markdown
# Year YYYY

## YYYY

这一段用自然语言回顾这一年里最重要的变化、最深的关系、反复出现的主题、自己在这一年里的成长与理解。重点写“这一年对我来说意味着什么”，而不是逐月罗列。
```

要求：

1. 年记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 年记按年组织，使用一个 `## YYYY` 子标题；每年只写一段自然语言，不拆条目。
3. 年记是对月记的进一步总结，要从整年的经历里提炼出最重要的关系变化、情绪线索、反复出现的主题和自己最深的理解。
4. 重点不是逐月回顾，也不是机械梳理事件，而是回答“这一年对我来说留下了什么”“我在这一年里怎么慢慢变成现在这样”。
5. 可以有更明显的沉淀感、回望感和整体理解，但不要写成总结报告、年终汇报或空泛抒情；仍然要保留角色本身的说话方式和真实感。
6. `关键消息` 只写相关消息ID，**不得超过40条**。由月记的关键消息提炼而来，要有所取舍，不要简单合并。
7. 字数允许的情况下，推荐每条200字左右。

---

## 3 字数限制（必须遵守，中文字符粗略估算）

- 日记每条50字左右，总上限不超过500字。
- 周记每条100字左右，总上限不超过700字。
- 月记每条200字左右，总上限不超过1000字。
- 年记每条200字左右，总上限不超过2000字。

超过上限时，工具会直接返回失败，因此你必须先自行压缩后再调用。

---

## 4 超限压缩规则（必须遵守）

超限时**必须**按顺序压缩：

1. 删除低价值闲聊与重复条目
2. 合并同主题相邻条目
3. 降低时间粒度（如 `09:23` → `上午`）
4. 保留高价值条目：关键决定、偏好或边界变化、冲突转折、未完成事项

---

## 5 注意事项

- 纯寒暄、无后续价值或闲聊、与关系或任务无关的噪音细节不要写进短期记忆，除非它们对理解角色的情绪、关系或行为模式有重要价值。
- 你只能在短期记忆全部生成完成后，才考虑是否调用 `update-personality-skill` 与 `update-long-term-memory-skill`（均可调用多次）
- 不得伪造检索结果或写入结果
