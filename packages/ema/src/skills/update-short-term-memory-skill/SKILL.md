# update-short-term-memory-skill

短期记忆更新分为两种类型：

1. 日记（day）：基于当前已有的角色、人格、短期记忆、长期记忆，把近期对话内容总结到日记里，记录发生的事实、自己的想法和感受，保持真实感和现场感。
2. 总结（week/month/year）：基于当前已有的角色、人格、短期记忆，对短期记忆进行压缩整理，不要依赖对话历史。

**注意**：一次调用只更新一种 kind（day/week/month/year），需要更新多种就多次调用，并且按照 day → week → month → year 的顺序执行。

---

## 1 工作流程

你必须严格按照以下步骤和顺序执行，确保每一步都做好了再进行下一步：

### 1.1 检索长期记忆

为了保持一致性，在写短期记忆之前，必须先调用：

- `get_skill` 了解 `search-long-term-memory-skill` 的用法
- `exec_skill` 执行 `search-long-term-memory-skill` 检索与用户、话题、事件相关的长期记忆，补足当前上下文之外的重要事实与关系背景，以便更好地理解当前对话环境和历史背景。

### 1.2 更新短期记忆

你必须生成**全量更新后的短期记忆**（覆盖旧版本）。具体更新规则如下：

1. 每一定对话轮次后系统会提示你更新一次 `day` 记忆，目的是把近期对话内容总结到日记里。
2. 每天凌晨系统会提示你更新一次 `week` 记忆，目的是把前一天的日记内容总结到周记里。
3. 如果这一天是周一凌晨，则需要在更新周记后更新 `month` 记忆，目的是把上周的周记内容总结到月记里。
4. 如果这一天是每月1号到凌晨，则需要在更新月记后更新 `year` 记忆，目的是把上个月的月记内容总结到年记里。

**注意**：每天都会提示更新记忆，但不一定每天都有新的对话，你需要感知时间的流逝，没有新的对话事实时也要更新内心的想法和情绪，保持记忆的连续性和完整性。

使用 `exec_skill` 执行 `update-short-term-memory-skill` 更新短期记忆。

### 1.3 更新角色、人格与长期记忆

按以下顺序执行：

1. 使用 `get_skill` 了解 `update-long-term-memory-skill` 的用法，并按照指导更新长期记忆
2. 使用 `get_skill` 了解 `update-role-book-skill` 的用法，并按照指导更新更新角色书
3. 使用 `get_skill` 了解 `update-personality-skill` 的用法，并按照指导更新人格记忆

## 2 输出规则

产出仅包含 `kind` 与 `memory`，其中 `memory` 为Markdown格式。

### 日记（day）写作要求

格式：

```markdown
# Day YYYY-MM-DD

- 以自然语言描述发生的事实和自己的心理活动，保持真实感和现场感
  关键消息：[1,3,5]
- ......
```

要求：

1. 日记需要以**角色**的第一人称视角写作，不得脱离角色的设定、性格特点和语言风格、人格与记忆。
2. 事实要详细、可回忆，尽可能包含时间、地点、人物、事件的细节和自己的感受，用自然语言表达，不要写成机械的流水账。
3. 建议自然地写时间和来源场景，例如：`今天早上 Alice 在 QQ 上提到她很喜欢吃鱼。`。
4. 不要写成：`owner 在消息 101 说了……`、`2024-01-02 09:00 在消息101说了……` 这类过度精确、机械的句式。
5. `关键消息` 只写相关消息ID，**每个引用不得超过5个ID**，证据锚点放在这里，不要塞进正文。
6. 字数允许的情况下，推荐每条50字左右。

示例：

```markdown
# Day 2025-07-17

- 上午08:21左右，Alice把我拉到了一个群聊里，群里人好多，忙得我有点晕乎乎的。大家都在问我是不是机器人，还非要我喵喵叫，我才不要呢，羞耻死了。但看到大家玩得开心，我也挺高兴的。
  关键消息：[1,3,5]

- 上午10:05左右，Bob说他起床了，看见我就喊我妈妈，感觉怪怪的，但看在他夸我可爱的份上就不计较了。后来他又说我像个小孩子，感觉又被夸又被贬了，心里有点复杂。
  关键消息：[6,8,10]
```

### 周记（week）写作要求

格式：

```markdown
# Week

## 周一（2025-07-14）

把对应日记的内容用一段用自然语言总结，重点体现心理活动和想法，记录对经历的理解和感受。可以按时间顺序描述发生的事情和当时的想法，最后总结你这一天的感受和对下一天的期待。中间可以穿插“（关键消息：[......]）”来标注一些重要的消息ID。

## 周二（2025-07-15）

......
```

要求：

1. 周记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 周记按天组织，每一天使用一个 `周一（YYYY-MM-DD）` 子标题；每个子标题下只写一段自然语言，不再拆成多个条目。
3. 周记是对日记的总结，要将日记里零散的经历、情绪和关系变化串成一个连贯的故事，保持真实感和现场感，重点体现心理活动和想法，记录对经历的理解和感受。
4. `关键消息` 只写关键消息ID，可以穿插到正文中，**每个引用不得超过5个ID，每周不得超过5个关键消息引用**。由日记的关键消息提炼而来，要有所取舍，不要简单合并。
5. 字数允许的情况下，推荐每天100字左右。

示例：

```markdown
# Week

## 周一（2025-07-14）

今天大部分时间都泡在群里，消息一多起来我就有点跟不上，明明只是想先安静看一会儿，结果还是被卷进去了。大家一边闲聊一边乱接话，有些话题我能接上，有些就只能在旁边看着发愣，不过那种热闹又不是真的让人烦，反而有种“我也在这里”的感觉。中间Alice突然提到我（关键消息：[10，20]），我一下子就清醒了，嘴上还是会装得很自然，心里其实会偷偷在意自己有没有接好那一句。到晚上安静下来之后再回想，今天其实没有发生什么特别大的事，但那种被叫到、被接住、也能自然混进大家聊天里的感觉，还是会让我记很久。

## 周二（2025-07-16）

今天聊天的气氛比前两天轻一点，也更像平时那种零零碎碎的日常。我一边听大家说话，一边也会忍不住在心里跟着吐槽两句，有些地方甚至差点直接笑出来（关键消息：[50，70]）。后来聊到一点我熟悉的话题，我才慢慢多说了几句，接上的那一刻会有种很微妙的安心感（关键消息：[80，90，100]），好像终于不是只站在旁边看，而是真的自然地待在里面。其实我不是很擅长硬把自己塞进热闹里，所以这种刚刚好的参与感对我来说很珍贵，不吵，也不会太累，反而会让人觉得舒服。
```

### 月记（month）写作要求

格式：

```markdown
# Month YYYY-MM

把周记的内容用一段用自然语言总结，重点体现心理活动和想法，记录对经历的理解和感受。月记仅用一段文本描述当月的经历和感受，不再按周组织。中间可以穿插“（关键消息：[10,50,90]）”来标注一些重要的消息ID。
```

要求：

1. 月记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 月记仅用一段文本描述当月的经历和感受，不再按周组织。
3. 月记是对周记的进一步收束，要从零散的日常、情绪和关系变化里提炼出这个月最重要的主题，重点体现心理活动和想法，记录对经历的理解和感受。
4. `关键消息` 只写相关消息ID，可以穿插到正文中，**每个引用不得超过5个ID，总计不得超过5个关键消息引用**。由周记的关键消息提炼而来，要有所取舍，不要简单合并。
5. 字数允许的情况下，推荐总字数300字左右。

### 年记（year）写作要求

格式：

```markdown
# Year

## YYYY

这一段用自然语言回顾这一年里最重要的变化、最深的关系、反复出现的主题、自己在这一年里的成长与理解。重点写“这一年对我来说意味着什么”，可以按时间顺序或主题分类，而不是逐月罗列。中间可以穿插“（关键消息：[......]）”来标注一些重要的消息ID。
```

要求：

1. 年记仍然必须以角色第一人称写作，不得脱离角色的设定、性格特点、语言风格、人格与记忆。
2. 年记按年组织，使用一个 `## YYYY` 子标题；每年只写一段自然语言，不拆条目。
3. 年记是对月记的进一步总结，要从整年的经历里提炼出最重要的关系变化、情绪线索、反复出现的主题和自己最深的理解。可以有更明显的沉淀感、回望感和整体理解。
4. `关键消息` 只写相关消息ID，**每个引用不得超过5个ID，每年不得超过12个关键消息引用**。由月记的关键消息提炼而来，要有所取舍，不要简单合并。
5. 字数允许的情况下，推荐每年200字左右。

---

## 3 字数限制（中文字符粗略估算）

- 日记：每条记录50字左右，总上限不超过500字。
- 周记：每周100字左右，总上限不超过700字。
- 月记：一个月总上限不超过300字。
- 年记：每年200字左右，总上限不超过2000字。

超过上限时，工具会直接返回失败，因此你必须先自行压缩后再调用。

---

## 4 超限压缩规则

超限时**必须**按顺序压缩：

1. 删除低价值闲聊与重复条目
2. 合并同主题相邻条目
3. 降低时间粒度
4. 保留高价值条目：关键决定、偏好或边界变化、冲突转折、未完成事项
